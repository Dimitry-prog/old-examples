# Biome Configuration Guide

Этот проект использует две конфигурации Biome для разных режимов разработки:

## Production Configuration (`biome.json`)

Используется для production сборки и CI/CD пайплайнов. Характеристики:

- **Строгие правила**: Большинство правил настроены как `error`
- **Автоматическая организация импортов**: Включена (`organizeImports: "on"`)
- **Строгая валидация**: Все потенциальные проблемы рассматриваются как ошибки
- **Запрет console.log**: `noConsole: "warn"` для предотвращения попадания отладочного кода в продакшн

#### Использование:
```bash
bun run lint:prod      # Проверка кода в production режиме
bun run lint:prod:fix  # Проверка и автоматическое исправление в production режиме
bun run format:prod    # Форматирование кода в production режиме
```

## Development Configuration (`biome.dev.json`)

Используется во время разработки для более комфортной работы. Характеристики:

- **Мягкие правила**: Многие правила настроены как `warn` вместо `error`
- **Отключенная организация импортов**: `organizeImports: "off"` для предотвращения конфликтов с IDE
- **Разрешенные console.log**: `noConsole: "off"` для отладки
- **Менее строгая валидация**: Фокус на критических ошибках, предупреждения для стилистических проблем

### Использование:
```bash
bun run lint:dev      # Проверка кода в dev режиме
bun run lint:dev:fix  # Проверка и автоматическое исправление в dev режиме
bun run format:dev    # Форматирование кода в dev режиме
```

## Default Configuration

По умолчанию используется development конфигурация для удобства разработки:

```bash
bun run lint          # Использует development настройки
bun run lint:fix      # Использует development настройки
bun run format        # Использует development настройки
```

## Основные различия

| Аспект | Production | Development |
|--------|------------|-------------|
| Организация импортов | Включена | Отключена |
| Console.log | Предупреждение | Разрешено |
| Неиспользуемые переменные | Ошибка | Предупреждение |
| Стилистические правила | Ошибка | Предупреждение |
| Недостижимый код | Ошибка | Предупреждение |

## Рекомендации

1. **Во время разработки**: Используйте `biome.dev.json` для комфортной работы
2. **Перед коммитом**: Запускайте production линтер для проверки качества кода
3. **В CI/CD**: Используйте только production конфигурацию
4. **IDE интеграция**: Настройте IDE на использование development конфигурации

## Интеграция с IDE

### VS Code
Добавьте в `.vscode/settings.json`:
```json
{
  "biome.configPath": "./biome.dev.json"
}
```

### WebStorm/IntelliJ
В настройках Biome укажите путь к `biome.dev.json` для development режима.
