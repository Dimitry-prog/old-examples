# Changelog - Table Component

## Исправления группировки колонок

### Проблема
При сворачивании группы колонок заголовок группы:
1. Оставался на всю ширину таблицы, не соответствуя ширине видимых колонок
2. Начинался с начала таблицы, а не над своими колонками
3. Перекрывал колонки без группы (ID, Name)

### Решение
Реализована динамическая логика подсчета colspan для заголовков групп:

1. **Подсчет всех видимых колонок** - система проходит по всем видимым заголовкам в порядке их отображения
2. **Сегментация с учетом негруппированных колонок** - создаются сегменты для:
   - Колонок без группы (пустые ячейки в строке заголовков)
   - Колонок с группой (заголовок группы с правильным colspan)
3. **Правильное позиционирование** - заголовок группы располагается точно над своими колонками
4. **Автоматическое обновление** - при сворачивании/разворачивании группы colspan пересчитывается автоматически

### Пример работы

**До исправления:**
```
┌─────────────────────────────────────┐
│  Product Info (▶)                   │  ← Начинается с начала, перекрывает ID и Name
├────┬──────────┬──────────┬──────────┤
│ ID │   Name   │ Category │ Stock    │
```

**После исправления:**
```
┌────┬──────────┬──────────┬──────────┐
│    │          │ Product  │ Inventory│  ← Правильное позиционирование
│    │          │ Info (▶) │ (▼)      │
├────┼──────────┼──────────┼──────────┤
│ ID │   Name   │ Category │ Stock    │
 ↑                ↑
Пустые ячейки   Заголовок над своими колонками
```

### Технические детали

Функция `renderGroupHeaders()` теперь:
- Анализирует порядок видимых колонок в таблице
- Группирует последовательные колонки (с группой или без)
- Создает сегменты для каждого блока колонок:
  - `{ groupName: null, colSpan: N }` - для колонок без группы (рендерит пустую ячейку)
  - `{ groupName: "Group", colSpan: M }` - для колонок с группой (рендерит заголовок)
- Поддерживает несколько групп в одной строке заголовков
- Автоматически пропускает рендеринг строки, если нет ни одной группы
